{% load ui_tags %} {% help_items as items %} key-round | Vaults
  • Create — Click New vault in the toolbar. Choose a name, icon, and colour, then set a master password.
  • Unlock — Click a vault card and enter your master password to decrypt its contents in this browser session.
  • Favourites — Star a vault to pin it to the Favourites view in the sidebar.
  • Shared with me — Vaults that another user has granted you access to appear in the Shared with me view.
  • Delete — Open the vault settings and choose Delete vault (owner only, irreversible).
--- lock | Entries & Folders
  • New entry — Unlock a vault, then click + to add a login, note, or card.
  • Folders — Organise entries in folders using the folder panel inside a vault.
  • Favourites — Star individual entries to surface them quickly.
  • Copy & autofill — Click a field to copy it to the clipboard; browser extensions handle autofill.
  • Trash — Deleted entries move to trash and can be recovered or permanently removed.
--- shield | Security
  • Zero-knowledge — Your master password never leaves your device. The server stores only a one-way hash.
  • AES-256-GCM — All entry data (usernames, passwords, TOTP secrets) is encrypted in your browser before being sent.
  • Per-vault keys — Each vault has its own randomly-generated encryption key, independently locked by its master password.
  • Sharing — Vault keys are re-encrypted with the recipient's public key client-side before being shared — the server cannot read them.
  • Master password — Choose a strong, unique master password. There is no server-side recovery if you forget it.
--- code | API Access
  • Base URL: /api/v1/passwords
  • Authentication: Use your session cookie or API token.
  • Documentation: Full API docs at /schema/swagger-ui/.
